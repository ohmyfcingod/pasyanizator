# ПасьянизаторЭкстрактор

Внешняя обработка 1С (external data processor). Снимает **JSON-карту EAV-модели видов документов** —
механизм дополнительных реквизитов и сведений БСП — для последующей генерации физических
`Справочник.<Вид>`. Есть **UI-форма** (витрина видов) и вызов **кодом**.

## Состав файлов (EDT-src)

Формат выгрузки EDT, каталог `ExternalDataProcessors/ПасьянизаторЭкстрактор/`:

- `ПасьянизаторЭкстрактор.mdo` — метаданные обработки (ТЧ `ТаблицаВидов` + форма `Витрина`);
- `ObjectModule.bsl` — весь код: публичные методы + внутренняя логика;
- `Forms/Витрина/Form.form` + `Module.bsl` — UI-витрина (обзор видов + кнопки).

Рядом лежит готовый бинарник `ПасьянизаторЭкстрактор.epf` (режим совместимости 8.3.27, см. ниже).

## UI-форма (Витрина)

![Витрина экстрактора: выбор состава, каталог выгрузки, снятие карты](../../pictures/Экстрактор.png)

Форма `Витрина` — обзор видов и запуск снятия карты. Таблица `Выгружать / Вид / Реквизитов / Документов`
(сортировка по числу реквизитов — EAV-тяжёлые сверху) и настройки:

- **Обновить статистику** — заполняет таблицу (лёгкие запросы, по кнопке; при открытии таблица пустая);
- **флажок «Выгружать»** в строке — выбор состава: снимаются отмеченные виды; если не отмечено ни
  одного — снимаются все виды из таблицы;
- **команды таблицы** — «Отметить все» / «Снять все» / «Только с документами» (виды с документами > 0);
- **Порог документов** — при значении > 0 автоматически отмечает виды, где документов не меньше порога
  (0 — отметки вручную);
- **Каталог выгрузки** — поле с кнопкой выбора папки (по умолчанию `КаталогВременныхФайлов() + "ПасьянсКарты"`).
  Выбор папки и «открыть каталог» используют **расширение работы с файлами** — оно подключается
  асинхронно и с проверкой (веб-безопасно; если не подключено — путь вводится вручную);
- **Порог прореживания, %** — порог заполненности, ниже которого реквизит уходит в EAV;
- **Открыть каталог после выгрузки** — флажок: по успеху открыть папку с картой в проводнике;
- **Сформировать карту** — снимает JSON-карту по выбранным настройкам и показывает путь к файлу.

## Готовый `.epf` и сборка

- **Готовый бинарник:** `ПасьянизаторЭкстрактор.epf` — собран в **режиме совместимости 8.3.27**
  (1С:Документооборот 3.0.20), открывается в ДО3 напрямую (Файл → Открыть или через «Дополнительные
  отчёты и обработки»). Собран на платформе 8.5.1; для базы с иным режимом совместимости пересоберите
  из исходников (режим совместимости внешней обработки не должен быть выше, чем у конфигурации).
- **Пересобрать:** сборка внешней обработки в EDT (`build_external_objects`) или Конфигураторе
  (`/LoadExternalDataProcessorOrReportFromFiles` против базы-хоста — её режим совместимости и получит
  бинарник). Исходники **валидируются в пустом проекте** внешних обработок — прямых ссылок на
  прикладные типы нет.

## Публичный контракт

```bsl
Функция СформироватьКарту(ПараметрыВызова) Экспорт
```

`ПараметрыВызова` — `Структура` или `Соответствие` (необязательный; `Неопределено` = дефолты). Ключи:

| Ключ | Тип | Деф. | Назначение |
| --- | --- | --- | --- |
| `ИменаВидов` | Массив из Строка | пусто | наименования видов (регистронезависимо); пусто = автоподбор топ-N по числу доп. реквизитов; ненайденное имя → в `предупрежденияВыбора` |
| `МаксВидов` | Число | `3` | размер топа при автоподборе |
| `ПорогПрореживанияПроцент` | Число | `1` | порог заполненности в %, ниже которого реквизит/сведение уходит в EAV (ТЧ или РС) |
| `КаталогВыгрузки` | Строка | врем. каталог | куда писать JSON (создаётся автоматически); дефолт — `КаталогВременныхФайлов()`, **задавайте свой** |

**Возврат** — `Соответствие` (значения читаются по ключу `["…"]`, не через точку):

- `успех` (Булево);
- при неуспехе — `сообщение` (Строка, напр. `"Не выбрано ни одного вида (проверьте ИменаВидов)"`);
- при успехе — `файл` (Строка, полный путь JSON), `видов` (Число), `сообщение` (Строка,
  `"Карта: видов N, реквизитов M, предупреждений K"`).

**Побочный эффект:** пишет `ПасьянсКарта_<Метаданные.Имя>_<yyyy-MM-dd_HHmmss>.json` (UTF-8,
`schemaVersion 1.2`).

**Вызов:**

```bsl
Результат = ВнешниеОбработки.Создать(Путь, Ложь).СформироватьКарту(Параметры);
Сообщить(Результат["сообщение"]);
```

Второй экспортный метод — `ДанныеВитрины()` — лёгкий обзор видов (`Массив` структур
`Вид`/`Реквизитов`/`Документов`), без тяжёлых запросов статистики заполненности.

### Ключи выходного JSON

- `schemaVersion` — `"1.2"`;
- `предупрежденияВыбора` — Массив строк (ненайденные виды);
- `экстрактор` — версия, дата ISO, база, версияКонфы;
- `итоги` — `видовВыбрано`, `реквизитовТащим`, `реквизитовВEAV`, `сведенийВсего`, `сведенийТащим`,
  `сведенийВРС`, `предупреждений`, `коллизийРазрешено`;
- `виды` — Массив описаний (`вид`, `справочник`, `классификатор`, флаги, `дефолтныеПоля`, `реквизиты`,
  `остаютсяВEAV`, `допСведения`, `eavХвост`, …);
- `коллизии` — Массив (`уровень`, `вид`, `было`, `стало`, `правило`);
- `банкТипов` — Массив XML-имён типов (напр. `CatalogRef.ВидыДокументов`).

Нормативная схема карты — [`../../contract/ПасьянсКарта.schema.json`](../../contract/ПасьянсКарта.schema.json).

## Переносимость

Компилируется **без прямых ссылок на прикладные типы** (типы приходят строками запросов и через
`XMLИмяТипа`/`Метаданные`), работает под `УстановитьПривилегированныйРежим(Истина)` — прав ролей не
требует, валидируется в пустом проекте внешних обработок.

Во **время выполнения** завязан на модель **Документооборота 3.x**: `Справочник.ВидыДокументов` /
`ДокументыПредприятия` / `НаборыДополнительныхРеквизитовИСведений`,
`РегистрСведений.ДополнительныеСведения` / `НастройкиНумерации`, `Справочник.Организации`, общий модуль
БСП `ОбщегоНазначения`. На конфе без этих объектов методы упадут на первом запросе — запросную часть
надо адаптировать.

## Границы

Публикуется только код обработки. Справочники, которые **генератор** Пасьянизатора создаёт по карте,
несут в `<comment>` метку `Пасьянизатор из вида <uuid>` — её пишет шаблон генератора
`Catalog.mdo.tpl`, **не** экстрактор. Экстрактор эту метку только **читает**: при разрешении коллизий
имён она позволяет отличить уже сгенерированный «свой» вид от чужого объекта конфы. Конфигурация-донор
(1С:Документооборот 3.x) проприетарна и в поставку **не входит**. Комментарии в коде намеренно хранят
историю решений (ЗАРы) — см. [`../../docs/ЗАРы.md`](../../docs/ЗАРы.md).
